# Admin9 API 骨架项目安全审计维度

> 适用对象：Laravel 12 REST API 骨架项目（JWT + Spatie Permission + Service Layer）
> 审计范围：仅关注脚手架自身应保障的安全性，不涉及下游项目按部署环境定制的配置项（如 CORS 域名收窄、缓存后端持久化、按邮箱维度限流等）。
> 审计维度共 32 项，按领域分为 9 个类别。

---

## 一、认证与会话管理

1. **JWT 生命周期合理性** — 签发、过期（TTL）、刷新（Refresh TTL）配置是否合理
2. **Refresh Token 滥用防护** — 重放攻击、并发刷新、Token 复用场景
3. **密码策略强度** — 最小长度、复杂度要求（大小写、数字、特殊字符）
4. **密码重置流程安全性** — Token 有效期、单次使用保证、邮件链接安全性
5. **登录失败处理** — IP 维度速率限制、失败响应是否泄露用户存在性

## 二、授权与访问控制

6. **Permission 中间件覆盖完整性** — 所有路由是否都有权限守卫，有无裸露端点
7. **超级管理员保护边界一致性** — 所有 Service/Controller 是否一致守护 super-admin 不可变规则
8. **批量赋权越权升级** — 能否通过 role_ids 参数分配超出操作者自身权限的角色
9. **Mass Assignment 防护** — Model 的 `$fillable` / `$guarded` 是否严格限定

## 三、输入验证与注入防护

10. **SQL 注入防护** — 原始查询、Filter 类中的参数绑定是否安全
11. **XSS 攻击防范** — 输入验证正则覆盖度、API 输出编码
12. **CSRF 适用性确认** — 无状态 JWT 架构下 CSRF 保护的必要性与实现
13. **请求验证覆盖率** — 所有写入端点是否都有对应的 FormRequest

## 四、敏感数据处理

14. **密码存储安全** — 哈希算法选型、传输过程是否存在明文暴露
15. **JWT Secret 管理** — 密钥强度、生成方式、`.env.example` 是否提供占位
16. **环境变量保护** — `.env` 文件 `.gitignore` 覆盖、`APP_DEBUG` 生产默认值
17. **API 响应数据脱敏** — 响应中是否泄露 password hash、token、内部 ID 等敏感字段

## 五、日志与审计

18. **审计日志覆盖完整性** — 所有敏感操作（CRUD、权限变更、登录）是否都有审计记录
19. **日志信息泄露** — 日志中是否记录了密码、token 等敏感字段
20. **日志文件权限与存储安全** — storage/logs 目录权限、日志文件访问控制

## 六、错误处理与信息泄露

21. **异常处理机制** — BusinessException 转换、全局异常兜底是否完整
22. **生产环境错误脱敏** — `APP_DEBUG=false` 时错误响应是否安全
23. **堆栈跟踪暴露** — 异常堆栈是否对外泄露

## 七、数据库与数据一致性

24. **事务一致性** — 多步操作（角色同步、权限迁移、用户更新）是否正确使用数据库事务
25. **依赖版本锁定** — `composer.lock` 是否提交到版本控制

## 八、速率限制与抗滥用

26. **各端点限流配置合理性** — 登录、刷新、系统 API 的限流阈值是否合理

## 九、代码质量与部署安全

27. **测试覆盖率** — 认证、授权、超级管理员保护等关键路径是否有测试
28. **架构分层一致性** — Controller → Service → Model 分层是否严格遵守
29. **重复造轮子** — 是否存在手动实现了 Laravel/第三方包已提供的功能，是否充分利用框架能力
30. **命名规范与最佳实践** — 路由、控制器、方法、变量、数据库字段、权限等命名是否符合 Laravel 社区惯例
31. **CLI 命令生产环境访问控制** — `super-admin:create` / `reset-password` 等命令的生产环境防护
32. **Seeder 生产环境禁用机制** — 禁用逻辑是否可靠，能否被绕过
